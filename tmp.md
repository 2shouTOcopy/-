microSD 卡功能需求文档（整理版）

1. 文档背景

1.1 项目背景

为提升设备在方案存储、图片保存、训练图像管理等场景下的存储扩展能力，设备新增 microSD 卡存储能力。
microSD 卡作为可移除扩展存储介质，主要用于业务文件存储与管理；eMMC 作为设备内置基础存储介质，承载系统基础数据与兜底能力。

1.2 建设目标

本需求旨在实现以下目标：
	1.	支持 microSD 卡插入后的识别、挂载、状态展示与安全弹出
	2.	支持方案、图片、训练图像等业务数据在 microSD 卡中的存储与管理
	3.	支持 eMMC 与 microSD 卡联合读取与结果聚合
	4.	支持常见异常场景下的错误提示、日志记录与系统保护
	5.	支持通过界面对 microSD 卡执行格式化操作

1.3 存储定位说明
	•	eMMC：设备内置基础存储介质，用于系统基础数据及无 microSD 卡场景下的业务数据存储
	•	microSD 卡：设备可移除扩展存储介质，用于业务文件扩展存储
	•	当 microSD 卡在线且可写时，业务数据默认写入 microSD 卡
	•	当 microSD 卡被移除后，存储于 microSD 卡中的方案、图片、训练图像等文件不再在主界面中展示

⸻

2. 总体设计原则

2.1 读写原则
	1.	当 microSD 卡在线且文件系统合法、处于可写状态时，业务文件默认写入 microSD 卡
	2.	当 microSD 卡不可写、异常、空间不足或被移除时，系统严格报错，不自动降级写入 eMMC
	3.	eMMC 与 microSD 卡联合读取时，按目录扫描并合并结果返回
	4.	对于重复文件，当前按文件名进行去重；若文件名相同，则 eMMC 中的文件优先展示，屏蔽 microSD 卡中重复文件

2.2 数据治理原则
	1.	microSD 卡空间满后，禁止继续写入
	2.	不支持自动清理旧数据
	3.	不对方案、图片、训练图像做自动淘汰
	4.	所有清理、删除类操作均由用户主动触发

2.3 可移除介质原则
	1.	支持“安全弹出”流程
	2.	用户未执行安全弹出而直接拔卡时，系统应识别为异常移除
	3.	写入中途若发生拔卡，系统对当前写入任务执行强制中断
	4.	microSD 卡相关异常不得导致主业务进程崩溃

2.4 可观测性原则

系统需对 microSD 卡相关关键事件进行状态展示、错误提示和日志记录，便于用户理解与售后排查。

3. 功能需求

⸻

DR-1 存储介质挂载与状态管理

DR-1.1 插卡检测与自动挂载

DR-1.1.1 功能描述
系统应支持对 microSD 卡插入事件进行检测，并在检测到插卡后自动进行介质识别、文件系统校验与挂载处理。

DR-1.1.2 处理要求
	1.	检测到 microSD 卡插入后，系统自动触发挂载流程
	2.	挂载前需完成：
	•	介质存在性校验
	•	文件系统格式校验
	•	分配单元大小校验
	3.	挂载成功后，更新以下状态信息：
	•	挂载状态
	•	总容量
	•	剩余可用容量
	•	文件系统类型
	4.	挂载失败时，界面应显示异常状态并提示原因
	5.	需记录插卡、挂载成功、挂载失败等日志

⸻

DR-1.2 安全弹出机制

DR-1.2.1 功能描述
系统应提供“取出 SD 卡”或等效功能入口，支持用户在物理拔卡前执行安全弹出操作。

DR-1.2.2 处理流程
	1.	用户点击“取出 SD 卡”
	2.	系统进入“安全弹出中”状态
	3.	系统停止接收新的 microSD 卡写入请求
	4.	若存在正在进行的写入任务，执行强制中断
	5.	对已完成写入的数据执行必要的落盘与卸载处理
	6.	卸载成功后提示用户“可安全移除 SD 卡”
	7.	卸载失败时提示用户当前不可移除，并保留异常日志

DR-1.2.3 补充说明
	•	安全弹出成功后，microSD 卡状态应更新为“已安全弹出”或“离线”
	•	安全弹出后，系统不再访问 microSD 卡内容，直至再次插卡并重新挂载

⸻

DR-1.3 异常移除检测

DR-1.3.1 功能描述
系统应支持对未执行安全弹出情况下的直接拔卡行为进行检测。

DR-1.3.2 处理要求
	1.	识别到 microSD 卡异常移除后：
	•	立即中断后续所有 SD 卡相关读写请求
	•	将 microSD 卡状态更新为“异常移除”或“离线异常”
	•	记录异常移除日志
	•	向界面和业务层返回统一错误信息
	2.	当前正在执行的写入任务应立即失败并中断
	3.	不得因异常移除导致系统卡死、崩溃或主线程阻塞

⸻

DR-1.4 文件系统格式校验

DR-1.4.1 支持范围
microSD 卡仅支持以下格式要求：
	1.	文件系统类型：FAT32
	2.	分配单元大小（簇大小）：32KB
	3.	容量上限：最大支持 2TB
	4.	容量下限：当前按产品要求支持 至少 128GB 的卡，其他容量范围是否支持待后续补充验证结论
	5.	提供格式化功能，格式化目标格式为 FAT32 + 32KB 簇大小

DR-1.4.2 处理要求
	1.	非 FAT32 文件系统的卡，系统不允许进入正常读写状态
	2.	FAT32 但簇大小不为 32KB 的卡，系统不允许进入正常读写状态
	3.	不支持的卡应提示“不支持的 SD 卡格式”
	4.	格式不匹配时，应支持用户通过界面执行格式化
	5.	格式化前需进行二次确认，并提示格式化将清空卡内全部数据

DR-1.4.3 格式化功能要求
	1.	系统提供 microSD 卡格式化入口
	2.	格式化仅针对当前插入并已识别到的 microSD 卡
	3.	格式化成功后自动重新挂载并刷新状态
	4.	格式化失败时应提示失败原因并记录日志

⸻

DR-1.5 容量监控与状态统计

DR-1.5.1 状态信息
系统需维护并展示以下信息：
	•	microSD 卡挂载状态
	•	总容量
	•	剩余可用容量
	•	文件系统状态
	•	是否可写

DR-1.5.2 状态定义建议
界面至少应支持以下状态：
	•	未插入
	•	挂载中
	•	在线
	•	在线但不可写
	•	格式不支持
	•	空间不足
	•	安全弹出中
	•	已安全弹出
	•	异常移除
	•	挂载失败

DR-1.5.3 界面展示要求
通过界面实时显示 microSD 卡状态信息，至少包括：
	•	当前状态
	•	总容量
	•	剩余容量

⸻

DR-2 读写路由与策略控制

DR-2.1 写入目标选择策略

DR-2.1.1 默认策略
	1.	当 microSD 卡在线、格式合法且可写时：
	•	新建方案默认写入 microSD 卡
	•	图片默认写入 microSD 卡
	•	训练图像默认写入 microSD 卡
	2.	当 microSD 卡不可写、未挂载、异常移除、空间不足时：
	•	严格报错
	•	不自动切换写入 eMMC

DR-2.1.2 用户配置
支持用户通过界面选择保存路径，保存路径可配置为：
	•	microSD 卡
	•	eMMC

DR-2.1.3 失败策略
若当前用户选择的目标介质不可写，则本次操作直接失败，并返回对应错误提示。

⸻

DR-2.2 容量阈值拦截

DR-2.2.1 功能描述
在每次写入前，系统需检查目标介质剩余空间是否满足本次写入要求。

DR-2.2.2 处理要求
	1.	写入前执行剩余空间检查
	2.	若剩余空间不足，则阻断本次写入
	3.	返回明确错误提示，如：
	•	“SD卡已满”
	•	“空间不足”
	4.	不允许先写入再失败
	5.	不允许因空间不足生成残缺文件或半文件

⸻

DR-2.3 联合读取与去重策略

DR-2.3.1 数据聚合
SCMVS 或其他上层请求数据时，系统应同时遍历 eMMC 与 microSD 卡对应目录，并将结果聚合返回。

DR-2.3.2 去重规则
	1.	当前按文件名进行去重
	2.	若 eMMC 与 microSD 卡中存在同名文件，则以 eMMC 中文件为优先展示版本
	3.	microSD 卡中的重复文件在聚合结果中被屏蔽

DR-2.3.3 展示说明
	•	microSD 卡移除后，microSD 卡中的数据从聚合结果中消失
	•	对于方案文件，是否在界面上单独增加“SD卡方案”页签，当前待定，不纳入本阶段实现范围

⸻

DR-2.4 存储上限控制

DR-2.4.1 当前结论
	1.	microSD 卡空间满后直接禁止写入
	2.	不支持自动删除旧文件
	3.	不支持自动清理图片、方案、训练图像

DR-2.4.2 待后续扩展项
以下能力当前先不纳入实现，可作为后续扩展方向：
	•	按百分比限制方案目录容量
	•	按百分比限制图片目录容量
	•	按类型设置不同目录容量上限

⸻

DR-3 业务文件与目录管理

DR-3.1 训练图像文件夹管理

DR-3.1.1 功能描述
系统应支持对训练图像相关文件夹进行管理。

DR-3.1.2 支持能力
	1.	支持展示训练图像文件夹
	2.	支持删除训练图像文件夹
	3.	删除前需进行二次确认
	4.	删除失败时应提示失败原因并记录日志

DR-3.1.3 简化说明
当前先按简单设计，不扩展复杂权限控制、批量操作、回收站等能力。

⸻

DR-3.2 图片保存

DR-3.2.1 功能描述
系统应支持将运行过程中的图片保存到目标存储介质中。

DR-3.2.2 基本要求
	1.	支持保存到 microSD 卡或 eMMC
	2.	保存前进行目标路径合法性与剩余空间校验
	3.	保存失败时返回明确错误信息
	4.	图片保存中若 microSD 卡被移除，应立即中断本次写入

⸻

DR-3.3 保存路径与名称设定

DR-3.3.1 路径配置
	1.	支持用户配置保存路径
	2.	支持多级目录
	3.	用户配置路径时，应基于所选介质根目录进行管理
	4.	路径非法时应进行拦截并提示

DR-3.3.2 特殊字符
	•	是否支持特殊字符，当前待定
	•	现阶段建议研发预留路径合法性校验能力

DR-3.3.3 命名规则
系统应支持保存文件夹路径和名称设定。
文件命名、同名覆盖策略、时间戳拼接规则等细节，建议在后续详细设计阶段继续补充。

⸻

DR-4 异常处理机制

DR-4.1 写入中途拔卡处理

DR-4.1.1 功能描述
当系统正在向 microSD 卡写入数据时，若发生拔卡，系统应执行异常中断处理。

DR-4.1.2 处理要求
	1.	当前写入任务立即失败
	2.	当前写入流程被强制中断
	3.	界面提示写入失败
	4.	更新 microSD 卡状态为异常移除/离线异常
	5.	记录异常日志
	6.	不得因异常中断导致进程崩溃

⸻

DR-4.2 异常提示词规范

系统应统一 microSD 卡相关错误提示，至少包括以下标准提示：
	•	“SD卡写入失败”
	•	“SD卡已满”
	•	“不支持的SD卡格式”
	•	“SD卡未插入”
	•	“SD卡挂载失败”
	•	“SD卡已被异常移除”
	•	“SD卡格式化失败”
	•	“SD卡不可写”

后续建议在详细设计中进一步定义：
	•	错误码
	•	触发条件
	•	用户提示文案
	•	日志打印内容

⸻

DR-4.3 格式化异常处理

DR-4.3.1 处理要求
	1.	若格式化过程中发生失败，应提示“SD卡格式化失败”
	2.	格式化失败后，不应进入可写状态
	3.	若格式化后自动挂载失败，应提示挂载失败
	4.	需记录格式化开始、成功、失败等操作日志

⸻

DR-4.4 状态与日志记录

DR-4.4.1 功能描述
系统需记录 microSD 卡相关关键操作与异常事件日志。

DR-4.4.2 日志范围
至少包括以下内容：
	•	插卡
	•	拔卡
	•	安全弹出
	•	异常移除
	•	挂载成功
	•	挂载失败
	•	格式校验失败
	•	开始格式化
	•	格式化成功
	•	格式化失败
	•	写入成功
	•	写入失败
	•	空间不足
	•	删除文件夹成功/失败

4. 界面需求

4.1 状态展示

界面需展示 microSD 卡基础信息：
	•	是否在线
	•	总容量
	•	剩余容量
	•	当前状态

4.2 操作入口

界面至少提供以下入口：

	•	格式化 SD 卡
	•	选择保存路径
	•	删除训练图像文件夹

4.3 提示要求

对所有关键失败操作需弹出明确提示，包括但不限于：
	•	插卡不合法
	•	格式不支持
	•	空间不足
	•	写入失败
	•	格式化失败
	•	异常移除

5. 非功能需求

5.1 稳定性要求
	1.	microSD 卡异常不得导致系统崩溃
	2.	microSD 卡异常不得影响主流程继续运行
	3.	发生异常时，应能够恢复到可继续识别新插卡的状态

5.2 可维护性要求
	1.	关键状态流转应明确
	2.	关键错误码与提示语应统一
	3.	日志应可用于售后定位问题

5.3 可扩展性要求

后续应便于扩展以下能力：
	•	独立的 SD 卡方案视图
	•	不同类型文件的容量比例控制
	•	更复杂的目录规则和命名规则
	•	特殊字符支持策略
	•	更细粒度的异常提示与恢复策略

⸻

6. 当前明确结论

根据当前讨论，已确认如下：
	1.	SD卡不可写时，严格报错，不自动降级到 eMMC
	2.	SD卡可写时，默认写 SD 卡
	3.	拔卡后，SD 卡中的方案从界面消失；是否单独提供 SD 卡方案界面待定
	4.	联合读取去重当前按文件名处理
	5.	空间满后直接禁止写
	6.	不允许自动清理
	7.	提供格式化 SD 卡功能
	8.	路径支持多级目录，特殊字符策略待定
	9.	写入中拔卡时，当前任务强制中断
	10.	支持状态与异常日志记录
	11.	训练图像文件夹删除当前按简单设计实现

	8.	安全弹出时，对被中断任务的用户提示语细节

8. 验收建议

8.1 挂载与识别
	1.	插入 FAT32 + 32KB 簇大小的 microSD 卡，可正常识别并挂载
	2.	插入非 FAT32 卡，界面提示“不支持的SD卡格式”
	3.	插入簇大小不为 32KB 的 FAT32 卡，界面提示“不支持的SD卡格式”

8.2 写入策略
	1.	SD 卡在线可写时，新建方案默认写入 SD 卡
	2.	SD 卡不可写时，写入直接失败，不自动落到 eMMC
	3.	空间不足时，禁止写入并提示“SD卡已满”或“空间不足”

8.3 异常处理
	1.	写入过程中拔卡，当前任务立即失败
	2.	拔卡后状态更新为异常移除或离线异常
	3.	异常场景下系统不崩溃

8.4 格式化
	1.	支持格式化为 FAT32 + 32KB 簇大小
	2.	格式化成功后自动挂载
	3.	格式化失败时提示失败原因并记录日志

8.5 联合读取
	1.	同时存在 eMMC 和 SD 卡数据时，可完成聚合展示
	2.	同名文件去重时，eMMC 优先
